Election Software Demo

This is a demo for a simple election software package written in Python/Flask. The demo is a complete application minus a few features so the demo application can run out of the box. The demo application uses SQLite which I would never use in a production environment. SQLite is included in the Python Standard Library. sqlite3 — DB-API 2.0 interface for SQLite databases — Python 3.12.3 documentation This application consists of 4 Flask Blueprints in order to break the application up in manageable pieces. There is currently only 1 static directory. The application uses Python Logging and the demo application currently logs to the console only.

There are 2 types of users.

Administration users and voters.

The demo and the production system does not allow the voters to authenticate. After all voting is private. A randomly generated token is used by a voter to vote. That same token is stored in the voting system. The voter has to have a stored token and the token can only be used once. The demo system creates 100 tokens in a spreadsheet that can be used to vote. During voting the token is hashed keeping the voter’s votes private and eliminating the chance of one token used to vote more than once.


Administrators must authenticate. The security isn’t as good as it could be in the demo in order to keep things simple. There are 2 types of Administrators, Super Administrators and Election Administrators.

Super administrators can create and delete other super administrators plus create and delete Election administrators. The creation and deletion of administrators is the only purpose of the Super Administrator. There can only be 2 Super Administrators and a Super Administrator can’t delete themselves.

Election Administrators set up the election. The first item is to set up the date and times of the election. Once the date of the election is set up the Election Administrators Offices, Classes (Groups), and Candidates. Once the election starts Administrators cannot change any election set up data items.



Select the date by using the calendar ICON on the far right.
Use 24-hour clock or 12-hour clock plus AM/PM indicator.
There must be a date set or entry of election data will not occur.
Election data cannot be changed after the start datetime is reached.
Voting can only take place after Start Date and before End Date.
Election results only run after End Date.
